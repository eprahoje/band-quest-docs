# Start Screen and Onboarding (Planejamento)

## Problema
Definir a primeira experiência do jogador ao abrir o jogo e iniciar uma nova jornada.

## Objetivos
- Criar uma tela inicial clara.
- Guiar o jogador para começar uma partida.
- Reduzir atrito no primeiro contato com o jogo.

## Escopo inicial
- Layout da tela inicial.
- Acesso para nova partida.
- Entrada para continuar jogo, se existir.

## Non-goals (nesta fase)
- Tutorial obrigatório ou sequência de cutscenes introdutórias.
- Animação de personagens ou sprites — a animação de menu é somente de UI (CSS/JS).
- Qualquer tela de seleção que dependa de arte original ainda não produzida.

## Critérios de aceite
- A primeira tela é compreensível em poucos segundos.
- O fluxo inicial não depende de instruções externas.
- Nenhuma tela do fluxo depende de asset de arte além de ícones de biblioteca.
