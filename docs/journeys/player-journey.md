# Player Journey

This journey maps the intended player experience from first launch to the main gameplay loop.

The source Mermaid diagram lives in [docs/features/0005-start-screen-and-onboarding/diagrams/player-journey.md](../features/0005-start-screen-and-onboarding/diagrams/player-journey.md).

## Journey goals

- Reduce friction on the first launch.
- Make the path to gameplay obvious.
- Keep onboarding short and useful.
- Connect the early flow to progression, economy, and reputation.

See the feature diagram for the flowchart source.

## Interpretation

- The journey begins with clarity and minimal decisions.
- The first explicit player choice is the era selection in feature 0005.
- The core loop should always lead back to progression decisions.
